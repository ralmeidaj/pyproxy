LGPD Gateway — Arquitetura do Sistema

Middleware Laravel que conecta o Payproxy aos sistemas LGPD de cada município, sem criar dependência entre eles.

Adicionar um novo município requer zero alterações no Payproxy — apenas uma nova Strategy no Gateway e um Connector instalado no servidor do cliente.

Visão Geral

Como os sistemas se conectam

Componentes

Dois projetos, duas responsabilidades

LGPD Gateway — Ciberian

DatumGatewayService
Laravel 11 · PHP 8.3+

Roda na infraestrutura Ciberian. Recebe todas as requisições do Payproxy, resolve a Strategy correta pelo tenant_id, gerencia cache Redis e isola falhas entre clientes.

  • Strategy Factory — mapeia tenant_id → classe Strategy via config/datum_clients.php
  • Cache Redis — TTL configurável por tenant (padrão: 5 min); header X-Cache: HIT/MISS
  • Fault isolation — falha de um Connector não propaga para outros tenants
  • Audit log — tabela gateway_logs registra toda requisição
  • Alerta por e-mail — após 5 falhas consecutivas (configurável)
Datum Connector — cliente

WebAPI template
.NET 8 · instalado on-premise

Template fornecido pela Ciberian. Instalado no servidor de cada município. Lê ROT e RIPD diretamente do SQL Server do Datum e os expõe via REST — o Gateway nunca acessa o banco diretamente.

  • Autenticação — X-Api-Key, key gerada na instalação
  • Repository por versão — RotRepositoryV1, RotRepositoryV2 — atualização do Datum não quebra o Gateway
  • Allowlist de órgãos — orgao_id validado no appsettings.json; outros retornam 403
  • Deploy — Docker (recomendado) ou IIS (Windows Server)
Referência da API

Endpoints expostos ao Payproxy

Método Rota Auth Descrição
GET /api/rot?tenant_id={id} X-Api-Key Retorna itens do ROT do tenant — operações de tratamento de dados, base legal, DPO responsável.
GET /api/ripd?tenant_id={id} X-Api-Key Retorna relatórios de impacto (RIPD) do tenant — nível de risco, status (Rascunho / Finalizado / Assinado).
GET /api/health?tenant_id={id} X-Api-Key Status do Connector do tenant — online/offline, versão Datum, latência em ms.
GET /api/tenants X-Admin-Key Lista todos os tenants configurados com status de conectividade. Uso restrito à equipe Ciberian.

Headers de resposta

X-Cache
HIT — servido do Redis · MISS — consultou o Connector
X-Cache-TTL
Segundos restantes no cache para esta chave. Ex.: 247
X-Datum-Status
ok — resposta fresca · cached — Connector indisponível, usando cache · unavailable — sem dados
⚠

O Gateway nunca retorna HTTP 5xx ao Payproxy

Connector indisponível → Gateway usa cache (se existir) e retorna 200 com X-Datum-Status: cached. Cache expirado → retorna 200 com status: unavailable e array vazio. O Payproxy nunca é impactado pela infraestrutura de um município.

Processo

Como adicionar um novo município

Estimativa: menos de 1 dia de trabalho da equipe Ciberian. Nenhuma linha de código no Payproxy precisa ser alterada.

1

Criar a classe Strategy no Gateway

Implementar DatumStrategyInterface com os métodos getRotItems(), getRipdReports() e healthCheck(). A classe faz a chamada HTTP ao Connector do município.

// app/Services/Strategies/JaguariunaDatumStrategy.php class JaguariunaDatumStrategy implements DatumStrategyInterface { public function getRotItems(string $orgaoId): array { ... } public function getRipdReports(string $orgaoId): array { ... } public function healthCheck(): array { ... } }
2

Registrar em config/datum_clients.php

Mapear o tenant_id do Payproxy para a nova Strategy e as credenciais do Connector.

'3' => [ 'strategy' => JaguariunaDatumStrategy::class, 'url' => env('DATUM_JAGUARIUNA_URL'), 'api_key' => env('DATUM_JAGUARIUNA_KEY'), 'orgao_id' => env('DATUM_JAGUARIUNA_ORGAO_ID'), 'timeout' => 5, // segundos 'cache_ttl' => 300, // 5 minutos ],
3

Adicionar variáveis no .env

URL do Connector, chave de API e ID do órgão no sistema Datum do cliente.

DATUM_JAGUARIUNA_URL=https://datum-connector.jaguariuna.sp.gov.br DATUM_JAGUARIUNA_KEY=<chave-gerada-na-instalação> DATUM_JAGUARIUNA_ORGAO_ID=JAG-001
4

Instalar o Datum Connector no servidor do cliente

Fornecer o template .NET ao time técnico da prefeitura. Deploy via Docker ou IIS. A equipe da prefeitura configura a connection string do SQL Server e define a API Key.

# Via Docker (recomendado) docker run -p 8080:8080 \ -e ConnectionStrings__Datum="Server=...;Database=datum;..." \ -e ApiKey="chave-gerada" \ -e OrgaosPermitidos="JAG-001" \ ciberian/datum-connector:latest
5

Reiniciar o container do Gateway

Para o PHP OPcache carregar o novo código (cache de bytecode sem invalidação automática).

docker restart lgpd-gateway-app # Verificar que o novo tenant aparece no status curl -H "X-Admin-Key: sua-admin-key" \ https://lgpd-gateway.ciberian.com.br/api/tenants
Requisitos Não Funcionais

Desempenho, segurança e extensibilidade

⏰ < 200ms

Latência Gateway (p95)

Excluindo latência do Connector. Com cache ativo: < 20ms.

🔗 99,5%

Disponibilidade mensal

Indisponibilidade de Connector não é contabilizada como indisponibilidade do Gateway.

🔒 TLS 1.3

Transporte seguro

HTTPS obrigatório em todos os saltos. Requisições HTTP sem TLS são recusadas.

🔐 SHA-256

API Keys

Armazenadas como hash SHA-256 — valor completo não recuperável. Keys do Payproxy e admin com escopos separados.

🌎 0 dados pessoais

Privacidade por design

Gateway não armazena dados de contribuintes. ROT e RIPD são dados institucionais do DPO, não de titulares.

🚀 1 dia

Novo cliente

Onboarding de novo município em menos de 1 dia — Strategy + .env + Connector. Sem deploy de Payproxy.
